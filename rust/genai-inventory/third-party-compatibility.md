# T19.3 third-party scorer compatibility matrix

This matrix is the implementation audit for the 112 third-party rows in
`scorers.json`: 106 permissively licensed metrics execute through native Rust
family modules and the six Phoenix metrics are rejected under D23. No Python
runtime or third-party Python distribution is used by the Rust execution
artifact.

| Family | Reference pin | Verified license | Manifest rows | Native | D23 rejected |
| --- | --- | --- | ---: | ---: | ---: |
| DeepEval | `deepeval==4.0.7` | Apache-2.0 (wheel METADATA and tagged source) | 44 | 44 | 0 |
| Ragas | `ragas==0.4.3` | Apache-2.0 (wheel METADATA and tagged source) | 37 | 37 | 0 |
| TruLens | `trulens==2.8.1`, `trulens-providers-litellm==2.8.1` | MIT (wheel METADATA and tagged source) | 25 | 25 | 0 |
| Phoenix | `arize-phoenix-evals==2.13.0` | Elastic-2.0, incompatible | 6 | 0 | 6 |

The exactness oracle additionally pins the optional deterministic backends used
by the Ragas reference environment: `rapidfuzz==3.14.3`,
`sacrebleu==2.6.0`, `rouge-score==0.1.2`, and `datacompy==0.19.0`. They are corpus-generation tools
only and are not Rust runtime dependencies.

## Provenance and behavior boundary

- DeepEval algorithms, prompts, schemas, parser behavior, thresholds, and
  metric/session classification were inspected from the Apache-2.0
  `deepeval/metrics/` tree in the 4.0.7 wheel and v4.0.7 tagged source.
- Ragas algorithms, prompts, result types, embedding call shapes, and
  single-/multi-turn sample mapping were inspected from the Apache-2.0
  `ragas/metrics/` tree in the 0.4.3 wheel and v0.4.3 tagged source.
- TruLens feedback templates, structured-output parsing, score normalization,
  and provider call shapes were inspected from the MIT
  `trulens/feedback/` tree in the 2.8.1 wheel and trulens-2.8.1 tagged source.
- MLflow's wrapper mapping and metadata behavior come from the manifest's
  reference SHA `2a36d19898fe7cdd5f596d4992be7494159efd15`.
- Phoenix source and prompt assets were not copied or translated. The only
  implementation is the user-ratified D23 rejection, which names Elastic-2.0
  and points to Faithfulness, RelevanceToQuery, Correctness, Safety, or a
  custom instructions judge as appropriate.

## Corpus and replay

`crates/mlflow-genai/tests/fixtures/third_party_golden.json` records all 112
manifest rows, pinned versions/licenses, deterministic values, family adapter
requests/parsers, dynamic-name errors, and the Phoenix disposition. It is
generated by `generate_third_party_oracles.py`; the full `uv run --with ...`
command and fake credential are in that script. `tools/third_party_oracle.py`
regenerates into a temporary directory, requires a byte-identical diff, and
then runs the native Rust replay suites. Both scripts guarantee zero live LLM
or embedding calls.
