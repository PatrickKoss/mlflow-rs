# proto option, so they legitimately do not appear in the Rust ROUTE_TABLE
# (which is generated purely from proto options). Format: `METHOD PATH`, one per
# line, each with a one-line reason. See rust/tools/route_parity.py.
#
# Two categories:
#   (A) genai / out-of-scope routes — stay on the Python server per the plan
#       (§1 Non-Goals, §2.2 nginx routing table). Rust never serves these.
#   (B) in-scope but hand-crafted Flask routes — Rust WILL serve these, but as
#       hand-written handlers (not via the proto route table). Kept here so the
#       proto-parity diff stays clean; their Rust implementations are covered by
#       their own tests in later phases.
